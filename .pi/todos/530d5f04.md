# Phase 3: Orchestrator

The brain — ties tracker, workspace, and agent together. Manages the full lifecycle of an issue from claim to completion or retry.

## Goals
- Define typed events emitted by the orchestrator
- Implement exponential backoff for retries
- Build the main polling loop: poll → claim → dispatch → monitor → retry

## Components

### 3.1 Event System (`internal/orchestrator/events.go`)
Typed events that flow through the entire system (orchestrator → TUI, logs, etc.):

| Event Type | Payload |
|------------|---------|
| `EventPollStarted` | timestamp |
| `EventIssueClaimed` | issue ID, timestamp |
| `EventWorkspaceCreated` | issue ID, workspace path |
| `EventPromptBuilt` | issue ID, prompt length |
| `EventAgentStarted` | issue ID, PID, session ID |
| `EventTokensUpdated` | issue ID, tokens in, tokens out |
| `EventAgentOutput` | issue ID, output chunk |
| `EventAgentFinished` | issue ID, success/failure, error |
| `EventIssueCompleted` | issue ID |
| `EventIssueRetrying` | issue ID, attempt number, retry at |
| `EventBackoffQueued` | issue ID, attempt, retry at |
| `EventStallDetected` | issue ID, last event age |
| `EventTimeoutDetected` | issue ID, elapsed time |

- All events carry `Timestamp time.Time`
- Events are emitted via a channel (`EventChannel`)
- TUI subscribes to event channel for live updates

### 3.2 Backoff Logic (`internal/orchestrator/backoff.go`)
Exponential backoff with jitter for retries:

```go
type BackoffEntry struct {
    IssueID  string
    Attempt  int
    RetryAt  time.Time
    Error    string
}
```

Backoff formula:
- Attempt 1: retry in 30s
- Attempt 2: retry in 60s
- Attempt 3: retry in 120s
- ...exponential, cap at 240s (4 minutes)

Add jitter (±20%) to avoid thundering herd.

Operations:
- `CalculateBackoff(attempt int) time.Duration`
- `EnqueueRetry(issueID, errorMsg string, attempt int)` → compute retry time, add to queue
- `ReadyEntries() []BackoffEntry` → return entries where `RetryAt <= now`
- `Remove(issueID string)` → cancel retry (e.g., when issue is done)

### 3.3 State Management (`internal/orchestrator/state.go`)
Track running issues and their current state:

```go
type RunState struct {
    Issue       types.Issue
    Attempt     int
    Process     *types.AgentProcess
    Phase       types.RunPhase
    StartedAt   time.Time
    LastEventAt time.Time
    TokensIn    int64
    TokensOut   int64
    Error       string
}
```

- `Map[issueID]*RunState` — active runs
- Operations: `Add()`, `Update()`, `Remove()`, `Get()`

### 3.4 Main Loop (`internal/orchestrator/orchestrator.go`)
The core orchestrator — ties everything together:

```go
type Orchestrator struct {
    Config      config.WorkflowConfig
    Tracker     tracker.Tracker
    Workspace   workspace.Manager
    AgentRunner agent.AgentRunner
    Events      chan types.OrchestratorEvent
    State       *StateManager
    Backoff     *BackoffManager
}
```

#### Poll Tick (every `poll_interval_ms`)
```
1. FetchIssues() from tracker
2. ReconcileRunning() — check stalls, timeouts
3. DispatchBackoff() — retry ready entries
4. DispatchReady() — claim and start new issues
```

#### ReconcileRunning()
- For each active run in `State`:
  - Check `LastEventAt`: if no event in X seconds → stall
  - Check `StartedAt`: if running > `max_runtime` → timeout
  - On stall/timeout: `Stop()` agent process, call `EnqueueRetry()`, emit `EventStallDetected` or `EventTimeoutDetected`

#### DispatchBackoff()
- Get `ReadyEntries()` from backoff queue
- For each ready entry:
  - `ClaimIssue()` from tracker
  - `Create()` workspace (or reuse)
  - Build prompt from config template
  - `Start()` agent
  - Update `State`

#### DispatchReady()
- Skip if `len(State.running) >= Config.max_concurrency`
- Fetch unclaimed issues from tracker
- For each unclaimed:
  - `ClaimIssue()` → mark in_progress
  - `Create()` workspace
  - Render prompt from template
  - `Start()` agent runner
  - Add to `State`
  - Emit `EventAgentStarted`

#### On Agent Done (from Process.Done channel)
- If `err == nil`:
  - `UpdateIssueState()` → done
  - `Cleanup()` workspace
  - Remove from `State` and backoff queue
  - Emit `EventIssueCompleted`
- If `err != nil`:
  - Increment attempt
  - `EnqueueRetry()`
  - Emit `EventIssueRetrying`
