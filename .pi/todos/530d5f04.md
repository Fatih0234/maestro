# Phase 3: Orchestrator — Event System, Backoff, Main Loop

> **Detailed spec:** [docs/context/phase-3-orchestrator.md](./docs/context/phase-3-orchestrator.md)

The brain — ties tracker, workspace, and agent together. Manages the full lifecycle of an issue from claim to completion or retry.

## Goals
- Define typed `OrchestratorEvent` for high-level events (separate from `AgentEvent`)
- Implement exponential backoff with jitter for retries
- Build state management for active runs
- Build main polling loop: poll → claim → dispatch → monitor → retry
- Emit events for TUI consumption

## Components

### 3.1 Event System (`internal/orchestrator/events.go`)

### 3.2 Backoff Logic (`internal/orchestrator/backoff.go`)

Exponential backoff with ±20% jitter:

| Attempt | Delay |
|---------|-------|
| 1 | 30s |
| 2 | 60s |
| 3 | 120s |
| 4 | 240s (cap) |

Operations:
- `CalculateDelay(attempt int) time.Duration`
- `Enqueue(issueID, errorMsg string, attempt int) *BackoffEntry`
- `Ready() []*BackoffEntry`
- `Remove(issueID string)`
- `Get(issueID string) (*BackoffEntry, bool)`

### 3.4 Main Loop (`internal/orchestrator/orchestrator.go`)

```go
type Orchestrator struct {
    Config      *config.Config
    Tracker     tracker.IssueTracker
    Workspace   workspace.Manager
    AgentRunner agent.AgentRunner
    Events      chan types.OrchestratorEvent
    State       *StateManager
    Backoff     *BackoffManager
    ctx         context.Context
    cancel      context.CancelFunc
    promptTmpl  string
}
```

#### Poll Tick (every `poll_interval_ms`)
1. `FetchIssues()` from tracker
2. `ReconcileRunning()` — check stalls, timeouts
3. `DispatchBackoff()` — retry ready entries
4. `DispatchReady()` — claim and start new issues

#### ReconcileRunning()
- For each run: check `LastEventAt` for stall, check `StartedAt` for timeout
- On stall/timeout: `Stop()` agent, `Enqueue()` backoff, emit event

#### DispatchBackoff()
- Get `Ready()` entries
- Re-claim issue, reuse workspace, restart agent

#### DispatchReady()
- Skip if at `MaxConcurrency`
- Fetch unclaimed issues, claim, create workspace, start agent

#### monitorAgent()
- Goroutine per active agent
- Reads `proc.Events` channel
- Updates tokens, forwards output, detects completion

#### handleAgentDone()
- On success: mark done, cleanup workspace, remove from backoff
- On failure: enqueue backoff, emit retry event

## Verification
- Backoff delays follow formula with jitter
- `Ready()` returns only entries where `RetryAt <= now`
- `ReconcileRunning()` detects stalls (> StallTimeoutMs)
- `ReconcileRunning()` detects timeouts (> AgentTimeoutMs)
- `dispatchReady()` respects MaxConcurrency
- `dispatchReady()` skips issues already running or in backoff
- Agent events flow through to OrchestratorEvent emissions
- `handleAgentDone()` handles success and failure paths
- Graceful shutdown stops all agents and closes channels
- All 13 event types emitted at appropriate times
