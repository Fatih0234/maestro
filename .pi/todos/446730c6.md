# Phase 4: TUI

Visualize and interact with the running orchestrator using Charm Bubble Tea.

## Goals
- Bridge orchestrator events to TUI updates
- Render live state: running sessions, backoff queue, event log
- Support keyboard interactions (quit, scroll, refresh)

## Components

### 4.1 Event Bridge (`internal/tui/events.go`)
Connect orchestrator event channel to Bubble Tea's update loop:

- Subscribe to `orchestrator.Events` channel in a goroutine
- Convert `types.OrchestratorEvent` to TUI-friendly internal events
- Send updates via Bubble Tea's `tea.Msg` mechanism
- Handle channel close gracefully on shutdown

```go
// Bridge subscribes to orchestrator events and forwards to TUI
func (b *Bridge) Start(orchestratorEvents <-chan types.OrchestratorEvent, model *Model) {
    for {
        select {
        case event, ok := <-orchestratorEvents:
            if !ok {
                return // channel closed
            }
            model, _ = model.Update(OrchestratorMsg{Event: event})
        }
    }
}
```

### 4.2 TUI Model (`internal/tui/model.go`)
Main Bubble Tea model using Model/Update/View pattern:

#### Model Fields
```go
type Model struct {
    issues    []types.Issue        // all fetched issues
    running   map[string]*runEntry // currently active issues
    backoff   []types.BackoffEntry // retry queue
    stats     Stats                // aggregated metrics
    events    []string             // event log (last N lines)
    width     int
    height    int
    scrollPos int
    err       error
}
```

#### Update Function
Handle messages:
- `OrchestratorMsg` — update state from orchestrator event
- `tea.KeyMsg` — handle keyboard input
- `tea.WindowSizeMsg` — update dimensions
- `tickMsg` — periodic refresh (update "last seen" timestamps)

#### View Function
Render the full TUI layout:


### 4.3 Session Table (`internal/tui/table.go`)
Render the running sessions table:

- Columns: Issue ID, Title (truncated), Phase, Tokens, Last Event
- Sort by start time (oldest first)
- Highlight stalled/running sessions differently
- Auto-refresh "last seen" age every second

### 4.4 Event Log (`internal/tui/events_view.go`)
Render the scrolling event log:

- Show last 100 events
- Format: `HH:MM:SS IssueID message`
- Auto-scroll to bottom on new events
- `↑↓` arrow keys to scroll manually

## Key Bindings
| Key | Action |
|-----|--------|
| `q` / `ctrl+c` | Quit |
| `↑` | Scroll event log up |
| `↓` | Scroll event log down |
| `r` | Force refresh (trigger manual poll) |
| `esc` | Reset scroll position |
