# Phase 5: Polish & CLI

Ship it properly — a working, distributable CLI with graceful shutdown and basic test coverage.

## Goals
- Create the CLI entry point with all flags
- Implement graceful shutdown (signals, cleanup)
- Add basic test coverage
- Ensure everything integrates end-to-end

## Components

### 5.1 CLI Entry Point (`cmd/contrabass/main.go`)
Cobra-based command structure:

```bash
contrabass --config WORKFLOW.md [flags]
```

#### Flags
| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--config` | string | (required) | Path to WORKFLOW.md |
| `--dry-run` | bool | false | Exit after first poll cycle |
| `--log-file` | string | "contrabass.log" | Log output path |
| `--log-level` | string | "info" | Log level: debug/info/warn/error |
| `--no-tui` | bool | false | Headless mode — log events to stdout |
| `--port` | int | 0 | Web dashboard port (0 = disabled) |

#### Modes
- **TUI mode** (default): Run orchestrator with Bubble Tea UI
- **Headless mode** (`--no-tui`): Run orchestrator without UI, log events to stdout/stderr
- **Dry run** (`--dry-run`): Single poll cycle, print results, exit

#### Initialization Sequence
```
1. Load config from --config path
2. Initialize logger (file + console)
3. Create tracker (local board)
4. Create workspace manager
5. Create agent runner (OpenCode)
6. Create orchestrator
7. Start TUI or headless loop
```

### 5.2 Graceful Shutdown
Handle OS signals for clean exit:

#### Signal Handling
- `SIGINT` / `SIGTERM` → initiate shutdown
- Timeout: 30s max for graceful shutdown before force kill

#### Cleanup Operations (in order)
1. Stop accepting new issues
2. Signal all running agents to stop (`agent.Stop()`)
3. Wait for `Process.Done` channels (with timeout)
4. Force kill orphaned agent processes
5. Clean up workspaces (git worktree remove)
6. Close log file
7. Exit

#### Error Recovery
- Wrap main loop in `recover()`
- On panic: log stack trace, attempt cleanup, exit with code 1

### 5.3 Logging (`internal/logging/`)
Structured logging setup:

- Use `charm.land/log` for structured logging
- Log levels: debug, info, warn, error
- Outputs: log file + console (tty-aware, color in terminal)
- Log rotation: none for minimal version (keep simple)

Log messages should include:
- Timestamp
- Level
- Message
- Context fields (issue ID, phase, etc.)

### 5.4 Tests (`internal/*/_test.go`)
Basic test coverage across packages:

#### Config (`internal/config/config_test.go`)
- Parse valid WORKFLOW.md with all fields
- Parse WORKFLOW.md with defaults (missing optional fields)
- Error on malformed YAML front matter
- Error on missing required fields

#### Tracker (`internal/tracker/local_test.go`)
- Fetch issues from temp directory
- Claim and release cycle
- State transitions
- Error on non-existent issue

#### Workspace (`internal/workspace/manager_test.go`)
- Create and detect worktree
- Path resolution
- Cleanup (manual — needs git repo)

#### Orchestrator (`internal/orchestrator/backoff_test.go`)
- Backoff calculation (30s, 60s, 120s, capped)
- Backoff entry ordering
- Ready entries filtering

#### Agent Mock
For testing orchestrator without real OpenCode:
- `internal/agent/mock.go` — mock `AgentRunner` that simulates success/failure/delays
- Used by orchestrator tests

## Integration Testing
Manual end-to-end test:
1. Create a test issue in `.contrabass/board/issues/`
2. Run `contrabass --config WORKFLOW.md --no-tui`
3. Confirm: issue claimed → workspace created → agent started → completed
4. Check issue state is `done` after completion
