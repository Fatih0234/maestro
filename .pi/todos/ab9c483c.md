# Phase 1: Foundation — go.mod, Config Parser, Types, Tracker, Workspace

The infrastructure layer — no agent interaction yet. These components depend only on Go standard library.

## Goals
- Establish project dependencies
- Define shared types
- Read/write issues from file system
- Manage git worktrees

## Components

### 1.1 go.mod setup
- Initialize `go.mod` with module name
- Add dependencies: Bubble Tea v2, Lip Gloss v2, Cobra, Liquid (template rendering), fsnotify, charm log
- No version numbers yet — let `go mod tidy` resolve

### 1.2 Config Parser (`internal/config/`)
- Parse `WORKFLOW.md` with YAML front matter
- Define `WorkflowConfig` struct with fields:
  - `max_concurrency` (int, default: 3)
  - `poll_interval_ms` (int, default: 30000)
  - `tracker.type` (string, default: "internal")
  - `tracker.board_dir` (string, default: ".contrabass/board")
  - `tracker.issue_prefix` (string, default: "CB")
  - `agent.type` (string, default: "opencode")
  - `opencode.binary_path` (string, default: "opencode serve")
  - `opencode.port` (int, default: 0 = auto)
  - `opencode.password` (string, default: "")
  - `workspace.base_dir` (string, default: ".")
  - `workspace.branch_prefix` (string, default: "opencode/")
- Render prompt template using Liquid with `{{ issue.title }}`, `{{ issue.description }}`, `{{ issue.url }}` bindings
- Return `WorkflowConfig` and `promptTemplate string` from parser

### 1.3 Types (`internal/types/`)
Define shared types used across all packages:
- `IssueState` constants: `Unclaimed`, `Claimed`, `Running`, `RetryQueued`, `Released`
- `RunPhase` constants: `PreparingWorkspace`, `BuildingPrompt`, `LaunchingAgentProcess`, `InitializingSession`, `StreamingTurn`, `Finishing`, `Succeeded`, `Failed`, `TimedOut`, `Stalled`
- `Issue` struct: `ID`, `Identifier`, `Title`, `Description`, `State`, `Labels`, `URL`, `CreatedAt`, `UpdatedAt`
- `RunAttempt` struct: `IssueID`, `Phase`, `Attempt`, `PID`, `StartTime`, `TokensIn`, `TokensOut`, `SessionID`, `WorkspacePath`
- `BackoffEntry` struct: `IssueID`, `Attempt`, `RetryAt`, `Error`
- `AgentRunner` interface: `Start(ctx, issue, workspace, prompt) (*AgentProcess, error)`, `Stop(*AgentProcess) error`, `Close() error`
- `AgentProcess` struct: `PID`, `SessionID`, `Events (chan AgentEvent)`, `Done (chan error)`
- `AgentEvent` struct: event types and payloads from SSE stream

### 1.4 Local Board Tracker (`internal/tracker/`)
File-based issue storage under `.contrabass/board/`:
- `manifest.json` — schema version, issue prefix, next issue number, timestamps
- `issues/CB-1.json` — individual issue JSON files
- States: `todo`, `in_progress`, `done`
- Operations:
  - `FetchIssues() ([]Issue, error)` — list all non-done issues
  - `ClaimIssue(id) (Issue, error)` — mark as in_progress
  - `ReleaseIssue(id) (Issue, error)` — mark as todo
  - `UpdateIssueState(id, state) (Issue, error)` — update state
  - `GetIssue(id) (Issue, error)` — fetch single issue
- No external dependencies — pure file I/O

### 1.5 Workspace Manager (`internal/workspace/`)
Git worktree-based workspaces:
- Base path: `workspaces/` (configurable)
- Branch naming: `opencode/{issue-id}`
- Operations:
  - `Create(issue) (string, error)` — `git worktree add workspaces/CB-1 -b opencode/CB-1`
  - `Cleanup(issueID) error` — `git worktree remove workspaces/CB-1`
  - `Exists(issueID) bool` — check if worktree already exists
  - `Path(issueID) string` — return workspace path for issue
- Validate git repo exists before operations
- Handle git errors gracefully
