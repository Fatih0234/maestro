# Phase 1 Review Fixes

Fixes for issues found during Phase 1 code review.

## Priority Order

### 1. HIGH: Retry Queue State Preservation ✅
- **Option B** chosen: Store `retry_after` timestamp on issue, filter in `FetchIssues`
- Added `StateRetryQueued = "retry_queued"` as separate board state (Schema v2)
- Added `RetryAfter` field to both local Issue struct and types.Issue
- Added `SetRetryQueue()` method for convenient retry queue management
- Updated `FetchIssues()` to filter based on retry_after
- Added tests: `TestLocalTracker_RetryQueuedState`, `TestLocalTracker_SetRetryQueue`, `TestLocalTracker_UpdateIssueState_ClearsRetryAfter`

### 2. HIGH: Issue ID Not Sanitized in Branch Names ✅
- Added `sanitizeBranchName(id string)` function - allows `[a-zA-Z0-9_/-]`
- Added `sanitizeFileName(name string)` function - allows `[a-zA-Z0-9_.-]`
- Both functions replace invalid characters with hyphens
- Used in `Create()` for branch names and `workspacePath()` for directory names
- Added tests: `TestSanitizeBranchName`, `TestSanitizeFileName`, `TestManager_Create_WithSpecialChars`

### 3. HIGH: CreateIssue Not Atomic ⚠️
- Documented as limitation - write order remains: manifest first, issue file second
- This is a known limitation that would require more complex transaction handling
- Could be improved in future with lock-based approach

### 4. MEDIUM: Validate() Doesn't Check Inner Config Fields ✅
- Added validation for `OpenCode.BinaryPath` (must not be empty)
- Added validation for `Codex.BinaryPath` (must not be empty)
- Added tests: `TestValidate_EmptyOpenCodeBinaryPath`, `TestValidate_EmptyCodexBinaryPath`

### 5. MEDIUM: Cleanup Swallows Errors ✅
- Updated `Cleanup()` to handle orphaned worktrees
- When git reports "is not a working tree", attempts `os.RemoveAll()` to clean up directory
- Logs error if both git and directory removal fail
- Added test: `TestManager_Cleanup_NonExistentWorkspaceDirectory`

### 6. LOW: Missing IssueState.String() Method ✅
- `IssueState.String()` was already present in types/types.go (added in earlier commit)

### 7. LOW: runGit Error Output in Messages ⚠️
- Kept as-is: including git output in errors is acceptable for now
- Could add log-level based filtering in future if needed
